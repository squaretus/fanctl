# fanctl

Контроллер вентиляторов для MacBook Pro на Apple Silicon (M4). Терминальный TUI, MenuBar app и привилегированный XPC-демон.

Мониторинг температур и ручное управление скоростью кулеров через SMC (System Management Controller).

## Возможности

- Мониторинг всех температурных сенсоров (CPU, E-Core, GPU, батарея, SSD и др.)
- Управление скоростью всех кулеров одновременно через пресеты (0%, 25%, 50%, 75%, 100%)
- Цветовое кодирование температур (зелёный -> жёлтый -> красный)
- Автоматическое восстановление авто-режима при выходе
- Защита от «осиротевшего» состояния при аварийном завершении
- Три интерфейса: терминальный TUI, MenuBar app, CLI-утилита в PATH
- Привилегированный XPC-демон — управление кулерами без sudo
- Автообновление через GitHub Releases — проверка, уведомление, установка в один клик

## Установка

1. Скачать `FanCtl-x.x.x.pkg` из [Releases](../../releases/latest)
2. Открыть скачанный файл
3. macOS заблокирует запуск — это нормально, PKG не подписан Developer ID сертификатом

### Разрешение запуска

**Через системные настройки (рекомендуется):**

Системные настройки → Конфиденциальность и безопасность → внизу страницы появится сообщение о заблокированном файле → нажать **"Всё равно открыть"**

**Через терминал:**

```bash
xattr -d com.apple.quarantine ~/Downloads/FanCtl-*.pkg
```

После этого открыть PKG повторно.

4. Пройти визард установки, ввести пароль администратора
5. После установки: демон запущен, app готов, CLI доступен в терминале

### Использование после установки

```bash
# CLI (без sudo — через XPC-демон)
fanctl                               # TUI-дашборд
fanctl --read-only                   # Только мониторинг
fanctl --once                        # Однократный вывод
fanctl --update                      # Проверить и установить обновление

# CLI (с sudo — прямой SMC, обратная совместимость)
sudo fanctl
```

MenuBar app запускается автоматически при логине. Можно отключить в **System Settings > General > Login Items**.

## Управление

### CLI

| Клавиша | Действие |
|---------|----------|
| `1` | 0% (Авто) — вернуть все кулеры в авто-режим |
| `2` | 25% скорости |
| `3` | 50% скорости |
| `4` | 75% скорости |
| `5` | 100% скорости |
| `↑` / `↓` | Скролл списка «Остальные датчики» |
| `Q` | Выход (с восстановлением авто-режима) |

## Удаление

Три способа полного удаления (app, daemon, CLI, логи, кэши, разрешения):

### Через приложение

MenuBar → ⚙ (настройки) → кнопка **"Удалить FanCtl"** → подтверждение → готово.

### Через Finder

Перетащить `FanCtl.app` из `/Applications` в корзину. Daemon автоматически обнаружит удаление и очистит все остальные компоненты.

### Через терминал

Скопировать скрипт удаления из app и запустить:

```bash
cp /Applications/FanCtl.app/Contents/Resources/uninstall-helper.sh /tmp/
bash /tmp/uninstall-helper.sh
```

## Автообновление

Демон автоматически проверяет наличие новых версий через GitHub Releases API:

- **При запуске приложения** и **каждые 6 часов** — всегда свежая проверка через GitHub API
- **MenuBar**: баннер «Доступно обновление» с кнопкой «Обновить» — скачивает PKG и устанавливает автоматически
- **MenuBar**: ⚙ (настройки) → кнопка «Проверить обновления» — ручная проверка
- **CLI**: `fanctl --update` — проверка и установка обновления из терминала
- **CLI**: жёлтое уведомление над панелью управления в TUI-режиме

Обновление выполняется демоном с root-правами: скачивание PKG → silent install → перезапуск всех компонентов.
